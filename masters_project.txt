🎯 Overview
In this project, you will extend the probabilistic forecasting models studied in class (RNNs, LSTMs, Transformers) by replacing Gaussian outputs with a discrete probabilistic formulation.

Instead of modeling the next value as:

xₜ ~ N(μₜ, σₜ²)

you will discretize the signal into 128 bins and model:

xₜ ~ Categorical(p₁, …, p₁₂₈)

This transforms time series forecasting into a sequence modeling problem, closely related to how modern language models (LLMs) work.

🧠 Key Idea
After quantization, each value of the time series becomes a token.

Your model will learn to:

👉 predict the next token given previous ones

This is exactly the same principle used in language models:

time series → sequence of tokens

forecasting → next-token prediction

sampling → sequence generation

In this sense, you are building a mini language model for time series.

🚀 Objectives
By the end of this project, you should be able to:

Understand limitations of Gaussian uncertainty modeling

Implement discrete probabilistic forecasting

Use embedding layers for quantized signals

Train autoregressive models with cross-entropy loss

Generate forecasts by sampling sequences

Compare continuous vs discrete uncertainty representations

Forecasting

Evaluation

Compare with Gaussian models from the original notebook

Analyze prediction quality and uncertainty behavior

🔥 Why This Matters
Gaussian models assume:

a single peak

symmetric uncertainty

Discrete models can represent:

multiple possible futures

asymmetric uncertainty

complex, non-Gaussian distributions

👉 This often leads to more realistic uncertainty modeling

🌟 Bonus (Optional)

🧠 Takeaway
This project connects:

classical time-series forecasting

DeepAR-style probabilistic models

modern sequence modeling

👉 showing that forecasting can be viewed as generative sequence modeling, just like in LLMs.
